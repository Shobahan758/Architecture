Template Documentation

Nexora Architecture & Interior Design HTML5 Template

Thank you for purchasing Nexora. This documentation explains how to install, edit, customize, and prepare the template for a live website or client handover.

Quick Start

Getting Started

Nexora is a static HTML template built with Bootstrap 5, local fonts, local vendor files, custom CSS, and lightweight JavaScript. No build tool is required for normal editing.

Important: this is an HTML template, not a WordPress theme. It cannot be installed from the WordPress dashboard.

Setup

Installation Guide

  1. Unzip the downloaded package.
  2. Open index.html in a browser to preview the default demo.
  3. Choose the homepage and inner pages you want to use.
  4. Upload the selected HTML files and the full assets folder to your hosting server.
  5. Keep the folder paths unchanged unless you also update all related asset links.

Recommended Local Preview

You can preview the template by opening HTML files directly. For the most accurate browser behavior, run any simple local server from the project root.

python3 -m http.server 8000

Editing

Customization Guide

Change Logo Text

Search for the brand markup in each page header and footer:

<a class="navbar-brand" href="index-1.html"><span>Ne</span>xora</a>

Change Colors

Main colors are controlled in assets/css/main.css with CSS variables. Update these values to match your brand.

:root {
  --accent: #b78a54;
  --dark: #171512;
  --soft: #f4efe8;
}

Change Fonts

Fonts are loaded locally from assets/vendor/fonts/google-fonts.css. You can replace the font files and update the font-family rules in assets/css/main.css.

Change Images

Replace images inside assets/images/pages/. Keep the same file names for the fastest update, or change the image paths inside the related HTML/CSS files.

Styles

CSS Structure

File Purpose

All custom CSS files include section comments so you can find page-specific blocks quickly.

Scripts

JavaScript Structure

The main script file is assets/js/main.js. It includes clearly commented sections for:

  • Dark/light theme toggle
  • Sticky header and back-to-top button
  • Mobile menu behavior
  • Reveal animations
  • Counter animations
  • Project filtering
  • Hero background sliders
  • Testimonials, project carousel, team slider, and gallery movement
  • Demo form validation behavior
Forms

Contact, Newsletter, Cart, and Checkout Forms

All forms are static demo forms. The included JavaScript prevents page reload and adds a validation class for front-end preview only.

Before using the template on a live website, connect forms to your own backend, email service, CRM, or e-commerce platform.
Media

Images and Assets

Demo images are stored locally inside assets/images/. For production or client work, replace the demo images with your own licensed photography.

  • Page images are organized under assets/images/pages/.
  • Preview images are stored in the assets folder.
  • Vendor files are stored in assets/vendor/.
